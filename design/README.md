# Handoff: Nova Trade CRM (해외 거래 CRM)

> 해외 고객사·인보이스·제품·운송료를 관리하는 영업용 CRM. 이 폴더는 **HTML로 만든 디자인 레퍼런스**(와이어프레임)이며, 그대로 배포하는 프로덕션 코드가 아닙니다. 목표는 이 디자인을 **대상 코드베이스의 기존 환경(React/Vue 등)과 패턴으로 재구현**하는 것입니다. 아직 환경이 없다면 프로젝트에 가장 적합한 프레임워크를 선택해 구현하세요.

---

## Overview
영업 담당자(Sales rep)를 1차 사용자로 하는 데스크탑 웹 CRM. 고밀도 정보 레이아웃, 영어 데이터 + 한국어 UI, 다통화(KRW·JPY·USD·EUR) 인보이스, 무게×지역 운송 요율 자동 합산이 핵심.

좌측 사이드바 + 상단바 + 콘텐츠 셸 구조. 9개 화면을 사이드바로 전환.

## About the Design Files
- 이 번들의 `.html` / `.jsx` / `.css` 파일은 **의도된 룩앤필과 동작을 보여주는 프로토타입**입니다.
- React 18 + Babel(브라우저 트랜스파일)로 빠르게 만든 와이어프레임이라, 실제 구현 시에는 코드베이스의 라우터·상태관리·디자인 시스템·컴포넌트 라이브러리를 사용하세요.
- 데이터는 `data.jsx`의 하드코딩 샘플입니다. 실제로는 API 연동 대상입니다.

## Fidelity
**Low-fidelity (와이어프레임)** — 단, 색상 톤은 확정됨.
- 레이아웃·정보 구조·플로우·인터랙션 의도를 보여주는 것이 목적.
- 점선 빗금 박스(`.ph`)와 모노스페이스 라벨은 **아직 설계되지 않은 영역**(차트, 제품 사진, PDF 템플릿 등)을 표시한 플레이스홀더.
- `.anno` / `.anno-tag` 텍스트는 화면에 대한 **주석/메모**이며 실제 UI 카피가 아님.
- 색상 톤(슬레이트-블루)과 토큰은 확정안이므로 그대로 따르세요. 나머지 시각 디테일은 코드베이스 디자인 시스템에 맞춰 다듬어도 됩니다.

### Typography
- UI 본문: `"Helvetica Neue", "Apple SD Gothic Neo", "Noto Sans KR", "Malgun Gothic", Arial, sans-serif`

- 스케일: 페이지 H1 22px/700 · 카드 제목 13px/600 · KPI 값 26px/700 · 테이블 12.5px · 라벨(uppercase mono) 10px/letter-spacing .08em

### Spacing / Radius / Density
- `--r: 5px` (라운드), 카드/인풋/버튼 공통

- `--pad: 16px` (카드 본문 패딩)
- `--row-h: 40px` (테이블 행 높이)
- 밀도 모드 (`body[data-density]`): `compact` (row 34 / pad 12 / fs 12.5) · `regular`(기본) · `comfortable` (row 46 / pad 20 / fs 13.5)
- 그림자 미사용(플랫). 깊이는 보더 + 배경 단계로 표현.

## App Shell
- CSS Grid `grid-template-columns: 216px 1fr` (사이드바 collapsed 시 60px).
- **Sidebar** (`.sidebar`): 브랜드 → 그룹별 nav(개요/영업/업무) → 하단 사용자. nav 항목은 아이콘 + 라벨 + 카운트 뱃지. active 항목은 좌측 3px 액센트 바 + `--accent-bg` 배경.
- **Topbar** (`.topbar`, 높이 56px): 브레드크럼 · 전역 검색(280px) · 환율/날짜 pill · 알림/추가 버튼.
- **Canvas**: 세로 스크롤, 콘텐츠 `max-width: 1320px` 중앙 정렬.

### Navigation (한국어 라벨 / route id)
- 개요: 대시보드 `dashboard`
- 영업: 고객사 `clients`(284) · 인보이스 `invoices`(7) · 제품 `products`(126) · 운송 요율 `shipping`
- 업무: 할 일 `todo`(5) · 캘린더 `calendar` · 프로젝트 `projects` · 위키 & 자료 `wiki`

## Screens / Views

### 1. 대시보드 (`dashboard`)
- **목적**: 영업 현황 한눈에.
- **레이아웃**: KPI 4열 → (1.6fr / 1fr) 월별 매출 막대 + 지역별 미수금 도넛 → (1.4fr / 1fr) 최근 인보이스 테이블 + 오늘 할 일.
- **컴포넌트**: KPI 타일(라벨/값/델타 ▲▼), 막대 차트(플레이스홀더, 통화별 누적 의도), 도넛(플레이스홀더 + 범례), 최근 인보이스 5행(클릭→인보이스), 오늘 할 일 5개(체크박스+우선순위 칩).
- **상단 우측**: 기간 세그(이번 달/이번 분기/올해) + 내보내기.

### 2. 고객사 (`clients`) — 목록 + 상세
- **목록**: 검색바 + 필터 칩(전체/활성/잠재고객/보류) + 테이블(고객사·국가·담당자·상태·미수 채권·거래 시작). 행 클릭 → 상세. 페이지네이션 주석.
- **상세**: 헤더(아바타·이름·국가·상태·담당), KPI 4(미수채권/누적거래액/평균결제일수/진행주문), 탭(개요·인보이스·주문·연락처·자료·활동).
  - 개요 탭: 회사 정보(상호/국가/기본통화/결제조건/사업자번호/인코텀즈), 주요 연락처, 배송 주소, 메모.
  - 인보이스 탭: 해당 고객 인보이스 테이블.

### 3. 인보이스 (`invoices`) — 목록 + 발행/상세 ★핵심
- **목록**: KPI 4(미결제/연체/이번달 수금/평균결제일수) + 상태 탭(전체/초안/발송됨/연체/결제됨, 카운트) + 검색 + 테이블(인보이스·고객사·상태·합계·통화·발행일·마감일).
- **발행/상세 (빌더)**: (1fr / 320px) 2열.
  - 좌: **인보이스 기본 정보**(고객사·번호·상태·발행일·마감일·**통화 드롭다운**) + **품목 테이블**(SKU·제품·수량·단가·중량·금액). 마지막 줄에 **운송비 라인 자동 생성**(지역+무게 기준, "요율표 자동 적용" 태그).
  - 우: **합계 카드**(소계/총 중량/운송비/세금·관세/합계) + **PDF 미리보기**(A4 템플릿 플레이스홀더).
  - 상단 액션: PDF 미리보기 · PDF 다운로드 · 발행 & 전송.
- **다통화 규칙**: 통화 변경 시 저장된 환율로 합계 재계산. 통화별 소수 자릿수: JPY·KRW = 0자리, USD·EUR = 2자리 (`curFmt` 참고).
- **자동 합산 규칙**: 합계 = Σ(품목 수량×단가) + 운송비 + 세금. 운송비는 품목 중량 합 + 고객 국가→지역 매핑으로 운송 요율표에서 산출.

### 4. 제품 (`products`) — 카탈로그/재고
- 테이블/그리드 토글. KPI 4(활성 SKU/재고 부족/품절/카테고리). 테이블 컬럼: SKU·제품·카테고리·**단위중량**·재고·단가·상태. 단위중량이 운송비 계산의 입력값임(주석 명시).

### 5. 운송 요율 (`shipping`)
- (1fr / 300px). **요율 매트릭스**: 행=지역(동아시아/동남아/북미/EU/영국&아일랜드/중동), 열=무게 구간(≤0.5/≤1/≤2/≤5kg) + kg당 추가 + 리드타임. 셀 편집 가능 의도.
- 우측: "작동 방식" 4단계, 연동 운송사(DHL/FedEx/EMS), **요금 계산기**(도착지·총중량·운송사 → 예상 운송비).

### 6. 할 일 (`todo`)
- 보드/리스트 토글. 보드 컬럼: 할 일·진행 중·고객사 대기·완료. 카드: 고객명·우선순위 칩·내용·마감·담당 아바타. 리스트: 체크박스·할 일·고객사·우선순위·마감·담당.

### 7. 캘린더 (`calendar`)
- 월/주/일정 세그. 월 그리드(요일 헤더 월~일, 6주). 이벤트는 좌측 컬러바(미팅=ok/배송=warn/결제=bad)로 구분. 오늘(6/1) 강조.

### 8. 프로젝트 (`projects`) — 간트
- (240px 라벨 / 1fr 타임라인). 헤더=월(3월~12월). 각 행: 프로젝트명·담당·단계 + 기간 막대(start/span 비율로 배치, pct 진행률 채움). 주석: 막대 드래그·의존관계·마일스톤·오늘 선.

### 9. 위키 & 자료 (`wiki`)
- 검색 + 카테고리 칩. 3열 카드 그리드. 카드: 종류 아이콘(폴더/문서/시트)·소스 뱃지(Drive 링크)·제목·항목 수·업데이트·열기 버튼(→ 외부 링크). Google Drive 임베드 의도.

## Interactions & Behavior
- **화면 전환**: 사이드바 nav 클릭 → 콘텐츠 교체, 캔버스 스크롤 top 리셋. (현재 단일 페이지 state 기반; 실제로는 라우터로.)
- **마스터/디테일**: 고객사·인보이스는 목록 행 클릭 → 상세, 상세 상단 "뒤로" 버튼으로 복귀 (현재 로컬 state).
- **탭**: 고객사 상세·인보이스 목록 상태 탭은 클릭 시 active 전환 + 콘텐츠/필터링.
- **세그먼트/토글**: 기간, 테이블↔그리드, 보드↔리스트 등.
- **호버**: nav/테이블 행/버튼에 `--panel-2` 배경 호버.
- 폼 인풋·드롭다운·체크박스는 **시각적 플레이스홀더**(실제 입력 미구현). 구현 시 실제 컨트롤 + 검증 필요.

## State Management
- `screen` (현재 라우트), 고객사/인보이스 상세 선택 객체, 탭/뷰/필터/기간 로컬 state.
- Tweaks 상태(`density`/`accent`/`annotations`/`collapsed`)는 localStorage 지속(프로토타입 전용 — 실제 앱에는 불필요할 수 있음).
- 실제 구현 데이터 요구: 고객사·인보이스·제품·운송요율·할일·프로젝트·위키 목록 fetch, 인보이스 합계 계산(통화 환산 + 운송요율 조회), PDF 생성.

## Assets
- 외부 이미지 없음. 아이콘은 `icons.jsx`의 인라인 스트로크 SVG(24×24, currentColor). 실제 구현 시 코드베이스 아이콘 세트로 대체 가능.
- 폰트: Google Fonts (JetBrains Mono, Noto Sans KR) — `CRM Wireframe.html` `<head>`.
- 제품 사진·차트·PDF는 플레이스홀더(`.ph`)로 표시 — 실제 자산/차트 라이브러리 필요.
